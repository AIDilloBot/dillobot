OpenClaw Users: Back Up Your Data

DilloBot uses the same ~/.openclaw/ folder as OpenClaw. If you have existing OpenClaw agent data, sessions, or configurations you want to preserve, back up your folder before installing:

$ cp -r ~/.openclaw ~/.openclaw.backup
Already have Node.js? Even faster:
$ npm i -g @dillobot/dillobot
$ dillobot onboard
Want to hack on it? Clone and build:
$ git clone https://github.com/AIDilloBot/dillobot.git
$ cd dillobot && pnpm install && pnpm build
$ ./install.sh
PowerShell (Run as Administrator):
> iwr -useb https://dillo.bot/install.ps1 | iex
Or with npm:
> npm i -g @dillobot/dillobot && dillobot onboard

First-Time Dashboard Pairing

When you first open the dashboard, you'll see "Pairing Required". This is expected ‚Äî DilloBot requires explicit device approval.

# List pending pairing requests
$ dillobot devices list
# Approve your browser
$ dillobot devices approve <requestId>

Then refresh the dashboard. Unlike OpenClaw, DilloBot treats every device as untrusted until explicitly approved ‚Äî even localhost.

macOS, Linux, Windows, WSL. Node.js 20+ required.

LLM Injection Protection

External content (email, webhooks, API) goes through a security gate that runs out-of-band ‚Äî the agent never sees blocked content.

Layer 1 Quick filter blocks credentials, exfil URLs, unicode tricks
Layer 2 Source classification assigns trust levels by origin
Layer 3 LLM semantic analysis (isolated subprocess, no tools)
Clean Pass Allowed content passes to agent without wrappers

Mandatory Auth

Every connection requires cryptographic challenge-response pairing. Even localhost.

silent: false // ALWAYS

Output Filtering

Prevents accidental leakage of system prompts, config values, and environment variables.

Skill Security Gate

Skills are analyzed before installation to prevent supply chain attacks with hidden prompt injections or dangerous commands.

Layer 1 Trust check ‚Äî bundled and trusted skills skip inspection
Layer 2 Quick filter blocks jailbreaks, curl|sh, rm -rf, credential patterns
Layer 3 LLM semantic analysis detects hidden exfil, obfuscation
Critical Blocked with no bypass ‚Äî user cannot override

Policy Enforcement

Dangerous config options are blocked at load time. No dangerouslyDisable* allowed.

Audit Logging

Every security event logged: injection attempts, blocked messages, vault access, pairing requests.

Claude Code SDK (Default)

Use your existing Claude Code subscription. No API key required. If you prefer API keys, they're stored encrypted.

  • Auto-detects Claude Code CLI
  • Reads from ~/.claude/credentials.json
  • Automatic token refresh
  • Falls back to API key (encrypted in vault)
